##  Предварительные требования

Для того чтобы приложение работало корректно, на вашем компьютере обязательно должна быть установлена и запущена СУБД **PostgreSQL**. 

Бэкенд приложения будет сохранять в неё всех пользователей, данные администратора и другие сущности. Без установленной базы данных сервер выдаст ошибку подключения и упадет.

1. **Скачайте PostgreSQL** с официального сайта: [https://www.postgresql.org/download/](https://www.postgresql.org/download/) под вашу операционную систему.
2. При установке задайте пароль для суперпользователя `postgres` и запомните его.
3. Создайте пустую базу данных (например, через утилиту pgAdmin или консоль psql), имя которой вы укажете в настройках проекта.
4. Убедитесь, что служба PostgreSQL запущена на вашем компьютере.

##  Запуск проекта в режиме разработки

Для работы приложения необходимо открыть **три отдельных окна терминала** (или три вкладки в VS Code), так как сервер, клиент и разовый скрипт должны работать независимо и не блокировать друг друга.

### Шаг 1: Установка зависимостей и запуск Сервера (Backend)
В первом окне терминала перейдите в папку бэкенда. 

Команда `npm install` прочитает список в `package.json` и скачает из интернета все библиотеки, необходимые для работы сервера (Express для маршрутов, Sequelize/pg для связи с базой данных, пакеты для авторизации). Без этого сервер не сможет запуститься.

Затем запустите сервер в режиме разработки. При самом первом старте ORM автоматически подключится к вашему установленному PostgreSQL и создаст все нужные таблицы.

```bash 
cd backend
npm install
npm run dev
```

### Шаг 2: Создание учетной записи Администратора
Чтобы вы могли войти в систему под учетной записью администратора, необходимо создать первого пользователя в базе данных. Откройте второе окно терминала и запустите специальный скрипт.

Важно: запускайте этот шаг только после того, как сервер на Шаге 1 успешно запустился и создал таблицы в БД.

```bash
cd backend
node createAdmin.js
```
После того как скрипт выведет сообщение об успешном создании администратора, это окно терминала можно закрыть.

### Шаг 3: Установка зависимостей и запуск Интерфейса (Frontend)
Откройте третье окно терминала. Здесь мы настраиваем клиентскую часть, которую видит пользователь.

Команда npm install здесь скачает совершенно другой набор библиотек, необходимых именно для визуальной части (сам React, библиотеки для UI-компонентов, Axios для отправки запросов на бэкенд и сборщик проекта).

После установки запустите фронтенд-сервер:

```bash
cd frontend
npm install
npm run dev
```

Приложение автоматически соберется и откроется в браузере по адресу http://localhost:3000 (или http://localhost:5173).
